## 前情提要

也算是個 `Zed` 粉了，不久前他們頻道發了 [這個](https://youtu.be/RJlQ8RoHoUA)，玩得很花。

也就是在這時候，畫風突變了。

## 一切從 Github Copilot 啟發

![image](/images/AI-大廠算力競爭之薅羊毛-2026/0xMultiplier.png)

這讓我大驚失色，儘管這倆模型放在 6202 年算遠古生物了，但能免費給你用，那還是致命誘惑。

雖然這麼說不對，不是免費，錢是公司在繳。

{% note purple %}

儘管 `Github Pro` 單月 10 鎂（組織成員 19 鎂），在價格上或許不是那麼有競爭力，但你還會得到香香的 `Pro Badge` 喔。

{% endnote %}

我第一件做的事情就是讓它幫我把所有博客文章做摘要。
透過 `Zed Agent` 高級優化（跟我們大 `VSCode` 比），`4o`、`4.1` 在實作上已經很夠用了（我不是純血 `Vibe Coder`），而且完美的解決了我不用切窗切來切去的問題，全都在 `Zed` 內完成。

於是我就開始研究。
除了 `Github Copilot`，在 `token` 會越來越便宜，大廠要開始搶客的時代，還有沒有甚麼滿足我們客家精神的 `LLM Provider` 或是邪門歪道。

換句話說，就算你不是程序猿，拿 `Github Copilot` 當 LLM 基底使用也是沒啥問題。。。

## 破壞遊戲規則

`Open AI` 有 `ChatGPT` 這種訂閱服務，同時也有 `OpenAI API` 這種按 `token` 用量計費的模式。
也有 `Anthropic` 的 `Claude Pro` 之於 `Anthropic API`。

也就是分成訂閱制或用量計費，或是說，面向末端使用者或開發者。

要破壞的就是這層規則。

假設大廠掌握的算力是固定的，該吸引的就是訂閱制的使用者，而非按用量計費的開發者。
畢竟，他們比起開發者們好操作多了。
再加上，如果是開發者，他們懂得包一層再賺一手，吃的還是我的算力，我何不賺第一手。

{% note purple %}

你要知道，有些人 Steam 收藏庫滿滿遊戲不玩。
也有人網飛、迪士尼普拉斯訂了沒在看。

{% endnote %}

為了吸引大家都來玩、都來訂閱，通常就透過綁卡試用或免費額度的手段引誘消費者上門。

那是不是就可以鑽這個漏洞，以稍微有點專業人士的能耐，喬裝成末端使用者，享受免費額度帶來的快樂。

## 反代

反向代理，隱藏客戶連接的後端，由一個後端轉發需求。

講簡單一點，我們把所有的 `LLM` 服務集中在一個盒子。
我們使用的客戶端也只面向這個盒子，由它轉發對各家 `AI` 廠商的請求。

最後再把結果回傳給我們，返回的內容也能先分析或是偽裝成特定的格式，欺騙某些客戶端軟體。

這樣一來，我們就可以透過這個方式統一管理所有 `LLM` 服務，並且可以自由切換目標廠商。

{% note purple %}

雖然 AI 現在對話就是很粗暴的流式傳輸。
`API routing` 也就 `models`、`chat/completions`、`embedding` 等，沒啥複雜的。

{% endnote %}

我也不是第一個想到這事的，甚至有廠商專門用這個方法挖錢，像是 `Open Router` 這種叫做 `中轉模型供應商` 在賣的服務，屬於是賽博二房東了。

那我們何不把剛剛說的那些試用、免費配額聚集起來，用反代搞？
這次用的就是 [CLI Proxy API](https://github.com/router-for-me/CLIProxyAPI)。
算是介面完整、流程清晰，甚至不怎麼需要配置，只管有帳號懂怎麼登入就好了的超級懶人反代工具。

## CLI Proxy API

[Release](https://github.com/router-for-me/CLIProxyAPI/releases/latest) 頁面下載，[管理介面](https://github.com/router-for-me/Cli-Proxy-API-Management-Center/releases/latest) 也順便。

需要做的啟動流程也不複雜：
1. 本體解壓後，`config.example.yaml` 改名為 `config.yaml`
2. `config.yml` 內的 `remote-management -> secret-key` 是管理的密碼，記得給，他會幫你加鹽
3. 執行檔運行起來，管理介面給他開起來，密碼輸入後就可以了
4. OAuth 登入，看你有啥就登啥，如果怕被鎖帳，請登小帳
5. 對了，`API token` 記得改，不過不改的話也行
反正是本地運行，預設密碼 `your-api-key-1`/`2`/`3`

你看，這麼一來，我就有 27 個模型囉，而且他們 ***免費***。

![image](/images/AI-大廠算力競爭之薅羊毛-2026/cliproxyapi-dashboard.png)

必須提到 `Antigravity` 這個賽博活菩薩，優質企業 `Google` 的良心。
我是不會用你爆改的 `VS Code`，但我會用你送的模型。
***雖然，我大學的信箱不知道是被鎖了還是怎樣，`Antigravity` 憑證已經常駐 403 了。***
我公司帳號是還能用，但有人有興趣的話，詳細可以看 [這個](https://www.ithome.com.tw/news/174106)。

建議還是先在 `Antigravity` 本體內登入過，我就是太急了，直接從 `CLI Proxy API` 裡面登入。

教完了。

## 手機上的解決方案

`Zed` 增加 `LLM Provider` 在我這邊算基操了。
但你如果想在手機上也能有類似 `ChatGPT` 這種客戶端，這裡推薦一個叫 [Chatbox AI](https://github.com/chatboxai/chatbox) 的。
雖然有點陽春，但基本上能用。

下載開啟後，手機電腦在同網域，選 `Open AI` 或直接添加。

![image](/images/AI-大廠算力競爭之薅羊毛-2026/chatbox-provider.jpg?x500)

記得先在電腦上，不管是用甚麼方式確認自己的內網 IP。

之後就是把 API 主機輸入 `http://{內網IP}:8317`（前面的步驟如果沒亂改端口就是 `8317`）。
API 金鑰就是上面第 5 步設定的那個。
恭喜，你成功反代 27 個模型到手機上了。

![image](/images/AI-大廠算力競爭之薅羊毛-2026/chatbox-detail.jpg?x500)

之後你想咋用就咋用。

![image](/images/AI-大廠算力競爭之薅羊毛-2026/chatbox-chat.jpg?x500)

{% note purple %}

畢竟是陽春 `Agent`，如果需要聯網搜尋的功能，請記得到設定裡面調成免費的 `Bing Search`。
要客家，所有東西都一定要選免費的

{% endnote %}

## 下個結論

能薅就薅，隨時注意又有甚麼大善人，不過也請請注意各家免費配額的使用限制與條款變動。
這些大廠可不是慈善機構，像綁卡自動扣款這玩法已經被玩爛了。

不招搖、見好就收，這是基本素養。

回到我一貫的立場，我不願意跟大廠妥協。
我不會為了方便就把自己綁死在某家的生態系裡，今天用反代薅羊毛，本質上就是在拒絕成為任何一家的忠實韭菜。

也不知道 LLM 的泡影遲早會破，就跟之前的 VR、區塊鏈、Web3 一樣。
當資本發現投入的算力換不回等比的利潤，當模型的能力觸及天花板卻還在吹下一代有多強。
當越來越多人發現 AI 產出的東西終究需要人來收尾，這場遊戲就會開始洗牌。

現在各家燒錢搶市佔率的瘋狂，恰恰就是泡沫最大的時候，也正是我們這些客家人最該趁亂撿便宜的時候。

等潮水退了，該留下的技術自然會留下，該淘汰的喧囂也會安靜，屆時還站在那裡的，永遠是真正懂得東西怎麼運作的人。

{% note purple %}

所以，趁現在免費的時候多用用。
但千萬別忘了自己為甚麼要用，以及沒有它的時候你還剩下甚麼。

{% endnote %}
